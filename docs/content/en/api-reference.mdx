# API Reference

## Client Credentials Token Endpoint

### POST /api/client-credentials/token

Exchange client credentials for an access token.

**Base URL**: `https://your-demo-app.vercel.app`

**Endpoint**: `/api/client-credentials/token`

**Method**: `POST`

**Content-Type**: `application/json`

#### Request Body

```json
{
  "client_id": "string (required)",
  "client_secret": "string (required)"
}
```

#### Success Response (200 OK)

```json
{
  "access_token": "string",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "string"
}
```

#### Error Responses

**400 Bad Request** - Missing required parameters
```json
{
  "error": "client_id is required"
}
```

**401 Unauthorized** - Invalid client credentials
```json
{
  "error": "invalid_client",
  "error_description": "Client authentication failed"
}
```

**500 Internal Server Error** - Server configuration issue
```json
{
  "error": "UMS_BASE_URL not configured. Please set it as an environment variable."
}
```

#### Example Request

```bash
curl -X POST https://your-demo-app.vercel.app/api/client-credentials/token \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "your-client-id",
    "client_secret": "your-client-secret"
  }'
```

#### Using the Access Token

Include the access token in the `Authorization` header:

```
Authorization: Bearer <access_token>
```

## Notes

- This endpoint uses the UMS API internally
- Will be replaced with a public API in the future
- Tokens typically expire after 3600 seconds (1 hour)
- Store tokens securely and refresh before expiration
